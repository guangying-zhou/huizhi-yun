## Affected Pages/Routes
- `/` — 智能工作台：我的任务列表、今日动态流、快捷工时录入
- `/projects` — 项目列表：卡片+列表视图切换、创建项目
- `/projects/[...projectCode]` — 项目主页壳：加载项目上下文、导航、权限
- `/projects/[...projectCode]/board` — 项目看板：拖拽状态变更、WIP 限制
- `/projects/[...projectCode]/tasks` — 任务列表：筛选排序、批量操作、键盘快捷键
- `/projects/[...projectCode]/bugs` — 缺陷列表：创建/指派/跟踪
- `/projects/[...projectCode]/code` — 代码关联：GitLab 同步、commit 列表
- `/projects/[...projectCode]/timesheet` — 工时统计
- `/projects/[...projectCode]/members` — 成员管理
- `/projects/[...projectCode]/settings` — 项目设置（业务状态、编号前缀、通知规则）

## Key Interactions to Verify
- 加载包含 `/` 的 `projectCode` 项目 → 页面与 API 路由正常解析 on `/projects/[...projectCode]/*`
- 看板拖拽任务到不同列 → 状态变更 on `/projects/[...projectCode]/board`
- 批量选择任务 → 批量修改状态/优先级/指派人 on `/projects/[...projectCode]/tasks`
- 任务列表键盘快捷键（J/K、X、E、Enter）生效 on `/projects/[...projectCode]/tasks`
- 创建工作项（任务/缺陷）→ 自动生成 `project_code-序号` 编号 on `/projects/[...projectCode]/tasks`
- 手动触发 GitLab 同步 → `#project_code-123` commit 关联到任务 on `/projects/[...projectCode]/code`
- 从任务详情关联 Codocs 文档 → 仅验证 Aims→Codocs 跳转 on `/projects/[...projectCode]/tasks`
- 任务指派 → 企业微信通知触发 on 任务详情
- 添加/移除项目成员 on `/projects/[...projectCode]/members`
- 项目业务状态流转（draft→approval_pending→active→paused→completed→archived）on `/projects/[...projectCode]/settings`

## Edge Cases
- 拖拽任务到已达 WIP 限制的列 on 看板
- 访问 `group/project` 形式的 project_code 时，页面刷新和深链访问仍正常
- 批量操作选择 0 个或 100+ 个任务
- GitLab 同步时 commit message 无 `#project_code-数字` 匹配
- 项目 `project_code` 重复或非法时，保存被拒绝
- 企业微信通知发送失败（API 超时）不应阻塞主流程
- 查看空项目（无里程碑、无任务）的各页面空状态
- 并发创建工作项时编号不重复

## Critical Paths
- 登录 → 创建项目 → 设置 `project_code` → 添加成员 → 项目进入 `active` → 创建里程碑 → 创建任务 → 看板拖拽 → 完成任务
- 登录 → 进入项目 → 创建缺陷 → 指派 → 修复 → 关闭
- 登录 → 触发 GitLab 同步 → 查看任务关联的 commit
- 登录 → 打开智能工作台 → 查看我的任务/动态 → 快捷录入工时
