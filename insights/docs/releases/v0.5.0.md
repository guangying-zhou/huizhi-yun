# Release v0.5.0 (First Milestone)

Date: 2025-09-29
Tag: `v0.5.0`

## 1. Overview
First formal milestone covering domain/auth unification (Plan A), environment simplification, editor reliability, tri‑state publication model, and media observability.

## 2. Scope Breakdown
### 2.1 Multi-tenant Domain & Auth Unification
- Central host classifier (`hostClassifier`) drives platform-root / reserved-subdomain / tenant-subdomain / custom-domain logic.
- OAuth flow standardized: prepare → auth subdomain → callback (no hostfix redirect layer).
- Relay handler for custom domains sets dual cookies when required.

### 2.2 Environment Simplification
- Canonical public vars: `NUXT_PUBLIC_SITE_URL`, `NUXT_PUBLIC_PLATFORM_ROOT_DOMAINS`, `NUXT_PUBLIC_PLATFORM_RESERVED_SUBDOMAINS`, `NUXT_PUBLIC_STORAGE_URL`.
- Deprecated removed: `PLATFORM_ROOT_DOMAINS`, `NUXT_PUBLIC_PLATFORM_URL`, `NUXT_PUBLIC_R2_PUBLIC_URL`.
- Added env validation script (`pnpm run check:env`).

### 2.3 Editor & Publication Model
- Publication states expanded to: Draft Only / Website Active / Unpublished Changes.
- R2 directory semantics: `contents/{business}/` vs `unpublished/{business}/`.
- Editor safeguards: unsaved changes guard (breadcrumb + route leave), unpublished draft modal, explicit publish / unpublish actions.

### 2.4 Media Subsystem
- Listing enrichment: MIME inference, size, lastModified, stable IDs.
- Stats endpoint: aggregates totalFiles, totalSize, recentUploads.
- Efficient size retrieval via ListObjectsV2 + fallback HEAD.
- Public asset host unified with `NUXT_PUBLIC_STORAGE_URL`.

### 2.5 UI / Styling / Theming
- Preview/Header CSS variable consolidation.
- Layout/theme scoping to avoid global leakage.
- Removal of legacy color mode persistence & server artifacts.

### 2.6 Auth & Membership
- Role-aware redirects (OWNER/ADMIN vs USER) across platform, tenant, custom domains.
- Membership-based authorization normalization; legacy company naming fully migrated to business.

## 3. Notable Fixes
- Eliminated redirect loops & host mismatch in OAuth.
- Accurate media sizes (previously 0 KB due to HEAD limitations).
- Consistent tenant subdomain retention through auth cycle.
- Prevented user navigation data loss in editor flows.

## 4. Backward Incompatibilities
| Area | Change | Action Required |
|------|--------|-----------------|
| Env | Deprecated vars removed | Ensure `NUXT_PUBLIC_STORAGE_URL` set |
| Publication | 2 → 3 state model | Update any automation relying on old boolean published flag |
| API (earlier breaking) | /api/company → /api/businesses | Use businessId in payloads |

## 5. Recommended Follow-up (v0.6.0 Targets)
1. Custom domain login E2E regression matrix + automated test harness.
2. Media management: delete API, thumbnail generation, manifest or DB-backed metadata caching.
3. Content version history (R2 object versioning or manifest snapshots) + diff UI.
4. Initial commerce foundations (product/catalog schema, price model, order draft entity).
5. Observability: structured logs & minimal metrics (auth success rate, publish latency, media upload throughput).

## 6. Quality & Risk Notes
- Media size relies on R2 ListObjectsV2 accuracy; if future pagination scale >1000 objects per tenant, add pagination cursor to API.
- Publication state detection is path-based; concurrent publish/unpublish operations should later add a compare-and-swap or timestamp check.
- Env validator currently synchronous; consider integrating into CI pipeline (pre-build step).

## 7. Upgrade Guidance
For deployments upgrading to >= v0.5.0:
- Remove deprecated env vars.
- Add `NUXT_PUBLIC_STORAGE_URL` (maps to custom asset domain).
- Re-publish existing sites once to ensure `publishedAt` & `lastEditedAt` fields normalized.
- Optionally run membership repair scripts if migrating from legacy company tables.
