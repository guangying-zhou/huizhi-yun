## 一、项目背景

中小型软件企业在日常运营中普遍面临以下痛点：

1. **多系统割裂**：项目管理用 Jira、文档用飞书/钉钉、CRM 用 Excel，数据无法打通，重复录入严重
2. **管理成本高**：写日报、催进度、做报表等管理动作消耗大量时间
3. **缺乏全链路追溯**：从需求到交付缺少完整追溯能力，项目过程难以复盘
4. **AI 能力难落地**：企业虽有 AI 意愿，但缺少将 AI 融入日常工作流的平台支撑

当前正值 AI Agent 技术成熟、企业降本增效需求迫切、国产替代窗口期，多重因素叠加形成最佳入场时机。

## 二、项目目标

### 2.1 总体目标

构建面向 20~500 人规模软件企业的**企业作业与管理云平台（SaaS）**，以 AI 为核心驱动力，覆盖企业运营关键场景，实现全流程数字化、数据互联互通、AI 辅助决策。

- **内部使用**：提高公司整体运营效率，实现研发、营销、管理的数字化闭环
- **对外销售**：搭建可商用的 SaaS 平台，为中小型软件企业提供一站式管理解决方案

### 2.2 量化目标

| 指标 | 目标值 |
|------|--------|
| 覆盖企业核心作业流程 | ≥ 80% |
| 管理工作成果覆盖率 | 100%，过程可追溯、结果可验证 |
| AI 辅助提升工作效率 | ≥ 30% |
| 跨系统切换成本降低 | ≥ 50% |

### 2.3 关键绩效指标

| KPI | 关联模块 | 目标值 |
|-----|---------|--------|
| 任务按期完成率 | Aims | ≥ 85% |
| 文档协作活跃度（日活） | Codocs | ≥ 60% 员工 |
| 审批流程平均耗时 | Workflow / Console | ≤ 24h |
| 商机转化率 | Altoc | 提升 20% |
| 发票-到账-核销闭环率 | Finance | ≥ 95% |
| AI 建议采纳率 | 全平台 | ≥ 50% |
| 用户跨模块使用率 | 全平台 | ≥ 3 模块/用户 |

## 三、项目范围

### 3.1 产品模块

| 模块 | 代号 | 定位 |
|------|------|------|
| 平台控制面 | Platform | 租户、订阅、部署、License、策略包与应用治理 |
| 企业运行服务 | Console | 企业配置、目录、认证、凭证保险箱、集成配置、员工入口 |
| 汇智云·文档 | Codocs | 知识管理与文档协作平台 |
| 汇智云·项目 | Aims | 研发项目全生命周期管理 |
| 汇智云·营销 | Altoc | AI 辅助线索到现金全流程管理 |
| 汇智云·资产 | Assets | 全域全生命周期资产管理 |
| 汇智云·财务 | Finance | 发票、到账、核销、支出、费用与项目财务核算 |
| 汇智云·协同 | Align | 未来可选深度组织协同增强 |
| 流程引擎 | Workflow | 全平台工作流支撑 |
| 迁移兼容 | Account | legacy 目录、身份和项目注册表迁移源 |

### 3.2 技术架构

- **前端**：Nuxt 4（Vue 3）Monorepo 架构，各模块独立部署
- **后端**：Nitro Server + FastAPI（Python，代码分析服务）
- **数据库**：MySQL + Redis
- **存储**：阿里云 OSS
- **实时协作**：Hocuspocus + Y.js CRDT
- **代码集成**：GitLab API
- **认证**：Console OIDC + 企业 IdP（CAS SSO / LDAP / 企业微信等），迁移期兼容 Account legacy bridge

### 3.3 不含范围

- 移动端 App（本期仅 Web 端，响应式适配移动浏览器）
- 私有化部署方案（本期仅 SaaS 模式）
- 国际化/多语言（本期仅中文）

## 四、里程碑计划（PIVR 模型）

本项目采用**汇智 PIVR 项目管理生命周期模型**，作为产品研发类项目，四阶段映射如下：

| PIVR 阶段 | 里程碑名称 | 时间 | 核心目标 | 模式 |
|-----------|-----------|------|---------|------|
| 🟢 P - 规划POC | 平台架构与可行性验证 | 2026.01.01 - 2026.03.31 | 架构选型、技术验证，Account + Codocs 核心功能跑通 | 历史完成 |
| 🔵 I - 核心MVP | 核心业务闭环构建 | 2026.04.01 - 2026.06.30 | Platform + Console 基座、Aims + Workflow 上线，实现研发管理最小可行产品 | 当前推进 |
| 🟡 V - 商用MMP | 经营闭环与商业化就绪 | 滚动计划 | Altoc + Finance + Assets 打通合同、交付、资产、发票、到账、核销闭环 | 当前目标 |
| 🟣 R - 市场PMF | 数据驱动迭代优化 | 滚动计划 | Align 作为可选深度协同增强，经营分析与用户反馈闭环持续迭代 | 后续增强 |

---

- [x] 用户/部门/角色/权限管理
- [x] CAS SSO + LDAP 同步 + 企业微信/钉钉登录
- [x] 应用管理与 API 密钥管理

- [x] 文档分类、版本管理、分享、审阅发文
- [x] 文档模板、导出、回收站

**评审标准**：Account + Codocs 功能完整可用，团队内部试用通过

### 🔵 I 阶段：核心MVP（进行中）

**目标**：构建研发管理最小可行产品，实现核心业务闭环

- [x] 项目立项与生命周期管理
- [x] PIVR 里程碑与交付物管理
- [x] 工作项（需求/任务/缺陷）管理
- [x] 看板视图与双视图驱动
- [x] Workflow 流程引擎集成（审批流）
- [ ] GitLab 代码关联与提交同步
- [ ] 工时记录与统计
- [ ] 项目报表与度量

**评审标准**：Aims 模块支撑团队日常研发管理，项目全生命周期可闭环

### 🟡 V 阶段：商用MMP（经营闭环推进）

**目标**：补齐商业化功能，达到最小可销售产品标准

- [ ] 客户管理（CRM）与客户画像
- [ ] 商机管理与销售漏斗
- [ ] 产品与报价管理
- [ ] 合同管理与回款追踪
- [ ] 发票、到账、核销与项目财务核算（Finance）
- [ ] 产品主档、环境、资产与交付成果台账（Assets）
- [ ] 安全加固、权限精细化
- [ ] AI 合规检查：交付物与配置要求匹配度校验

**评审标准**：通过内部 UAT，具备对外演示和试用条件

### 🟣 R 阶段：市场PMF（后续增强）

**目标**：根据用户留存和活跃数据，快速迭代优化

- [ ] 深度组织协同（Align）
- [ ] 考勤管理（钉钉集成）
- [ ] 公告与通知管理
- [ ] 会议室预约
- [ ] AI 效能分析：基于 P-I-V 阶段真实速率提供进度预测
- [ ] 用户反馈闭环与数据驱动迭代

**评审标准**：核心用户留存率 ≥ 60%，跨模块使用率 ≥ 3 模块/用户

## 七、风险评估

| 风险项 | 等级 | 应对策略 |
|--------|------|---------|
| 团队人力不足 | 高 | AI 辅助开发提效；优先核心功能，分阶段交付 |
| 多模块集成复杂度高 | 中 | 统一事实源和稳定业务键，通过 API、Console service token 与 Foundation adapter 实现数据贯通 |
| 第三方服务变更（钉钉API等） | 中 | 抽象适配层，降低耦合 |
| 性能瓶颈（实时协作高并发） | 中 | Hocuspocus 水平扩展，Redis 集群 |
| SaaS 多租户数据隔离 | 低 | 本期以单租户模式运行，后续扩展多租户 |

2. **商业价值**：形成可复制的 SaaS 产品，面向中小软件企业销售
3. **技术沉淀**：积累 Nuxt 4 Monorepo + AI Agent 的全栈技术方案
4. **数据资产**：建立企业运营全流程数据体系，支撑 AI 辅助决策
