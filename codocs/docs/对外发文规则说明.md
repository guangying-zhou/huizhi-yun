# 对外发文规则说明

**适用模块：** `account`、`codocs`  
**适用范围：** 部门文档的“对外发文”场景  
**版本：** 1.1  
**日期：** 2026-03-30

---

## 1. 目标

为部门文档建立统一的“对外发文”资格判定、提交流程、审批规则、发布后执行规则、通知规则和归档规则，确保 `account` 与 `codocs` 两个模块口径一致。

---

## 2. 组织分类规则

`account.departments` 增加字段：

- `org_type`：机构类型
  - `department`：部门
  - `committee`：委员会
- `dept_category`：部门类别
  - `1`：行政
  - `2`：业务支撑
  - `3`：业务
  - `4`：核心管理
  - `NULL`：不适用，如委员会

当前分类口径：

- `GMO` 总经理办公室：`4` 核心管理
- `AF` 行政财务部：`1` 行政
- `HR` 人力资源部：`1` 行政
- `BA` 商务部：`2` 业务支撑
- `RD` 产品研发部：`2` 业务支撑
- `SDC` 交付中心：`2` 业务支撑
- `HFZX` 汇房智选部：`2` 业务支撑
- `SP` 软件项目部：`2` 业务支撑
- `MC` 营销中心：`3` 业务
- 各委员会：`org_type = committee`，`dept_category = NULL`

---

## 3. 资格规则

“对外发文”是资格规则，不是角色权限规则。

具备对外发文资格的组织：

- `org_type = department`
- `dept_category >= 2`

具体解释：

- `dept_category = 1`：无对外发文资格
- `dept_category = 2`：可发起对外发文，但需选择业务部门
- `dept_category = 3`：可直接发起对外发文
- `dept_category = 4`：可直接发起对外发文

委员会不使用 `dept_category` 参与本规则判定。委员会原有审阅逻辑保持独立，不属于本文重点范围。

---

## 4. 提交页面规则

发布页面中，当当前文档所属部门满足 `org_type = department` 且 `dept_category >= 2` 时，发布类型下拉框显示“对外发文”。

选择“对外发文”后，表单包含以下字段：

- `发送给`：必填
- `发文事由`：必填
- `是否需要加盖公章`：可选
  - 若勾选，则文件级别至少为“重要文件”
- `文件级别`：必填，单选
  - 一般文件
  - 重要文件
  - 关键文件

额外动态字段：

- 若发文部门 `dept_category = 2`
  - 显示“业务部门”选择
  - 候选范围为所有 `org_type = department` 且 `dept_category = 3` 的部门
  - 若可选业务部门只有 1 个，则自动带出，不再要求手选
- 若文件级别为“关键文件”
  - 显示“审批委员会”选择
  - 候选范围为所有 `org_type = committee` 的委员会

页面中同时展示流程预览。

## 5. 审批流程规则

### 5.1 通用节点命名

- 部门经理审核
- 业务部门经理审核
- 委员会审批（过半通过）
- 分管领导审核
- 上级领导审批

### 5.2 节点规则

#### 一般文件

适用于：

- `dept_category = 2`
- `dept_category = 3`
- `dept_category = 4`

流程：

1. 部门经理审核
   说明：若发起人本人就是部门经理，则该节点跳过
2. 若 `dept_category = 2`，增加“所选业务部门经理审核”
3. 分管领导审核
4. 审批结束

#### 重要文件

流程：

1. 部门经理审核
   说明：若发起人本人就是部门经理，则该节点跳过
2. 若 `dept_category = 2`，增加“所选业务部门经理审核”
3. 分管领导审核
4. 上级领导审批
5. 审批结束

#### 关键文件

流程：

1. 部门经理审核
   说明：若发起人本人就是部门经理，则该节点跳过
2. 若 `dept_category = 2`，增加“所选业务部门经理审核”
3. 所选委员会审批
   说明：采用过半通过规则，`通过人数 = floor(委员人数 / 2) + 1`
4. 分管领导审核
5. 上级领导审批
6. 审批结束

### 5.3 业务部门规则

当发文部门为：

- `dept_category = 2`
  - 必须选择业务部门
  - 审批链中增加“业务部门经理审核”
- `dept_category >= 3`
  - 不需要选择业务部门
  - 审批链中不增加该节点

### 5.4 审阅人解析规则

- 部门经理审核：当前发文部门 `manager_uid`
- 业务部门经理审核：所选业务部门 `manager_uid`
- 分管领导审核：当前发文部门 `leader_uid`
- 上级领导审批：当前发文部门上级部门的 `leader_uid`，若为空则回退到上级部门 `manager_uid`
- 委员会审批：所选委员会全部成员参与表决，按过半通过

## 6. 通知规则

### 6.1 提交时

提交审阅后，系统向当前第一节点审阅人发送企业微信通知。

### 6.2 节点推进时

每一节点通过后，系统向下一节点审阅人发送企业微信通知。

### 6.3 审批完成时

审批全部通过后，系统向发起人发送“审批通过，请确认发布”的企业微信通知。

### 6.4 一般文件附加通知

当“对外发文”且文件级别为“一般文件”时，在审批结束后，额外向上级领导发送企业微信通知，内容为“该一般文件对外发文审批已完成，请知悉”。

### 6.5 发布后盖章通知

当“对外发文”在提交时勾选“需要加盖公章”后，发起人确认发布时，系统向具备 `seal_admin` 角色的公章管理员发送企业微信通知，提醒办理盖章事宜。

### 6.6 盖章完成通知

公章管理员确认盖章后，系统向发起人发送“已完成盖章，请继续确认发送”的企业微信通知。

### 6.7 发送完成通知

发起人或具备发送权限的部门文档经办人确认发送后，系统向发起人发送“已完成发送登记”的企业微信通知。

## 7. 发布后执行规则

### 7.1 执行状态

“对外发文”确认发布后，`document_reviews.execution_status` 使用以下状态：

- `pending_seal`：待盖章
- `pending_send`：待发送
- `sent`：已发送

状态推进规则：

- 需要盖章：确认发布后进入 `pending_seal`
- 不需要盖章：确认发布后直接进入 `pending_send`
- 盖章完成后：进入 `pending_send`
- 发送完成后：进入 `sent`

### 7.2 确认盖章

确认盖章在协同文档中心中执行：

- 入口：`个人文档 / 协同文档`
- 按钮显示条件：
  - 文档为对外发文发布版本
  - `execution_status = 'pending_seal'`
  - 当前用户具备 `seal_admin` 角色

盖章登记字段：

- `盖章类型`：多选
  - 公章
  - 法人章
  - 财务章
  - 合同章
- `文档页数`：必填，正整数
- `备注`：可选

系统记录：

- 写入 `document_seal_records`
- 更新 `sealed_at`
- 更新执行状态为 `pending_send`

### 7.3 确认发送

确认发送在部门文档的“对外发文”页面执行：

- 入口：`部门文档 / 对外发文`
- 按钮显示条件：
  - 文档为对外发文发布版本
  - `execution_status = 'pending_send'`

发送登记字段：

- `发送人`：必填，下拉选择公司用户
- `接收人`：必填
- `联系电话`：必填
- `发送途径`：必填
  - 邮件
  - 企业微信
  - 微信
  - QQ
  - U盘
- `对方账号`：除 `U盘` 外必填
- `备注`：可选

发送权限：

- 发起人可确认发送
- 发文部门具备部门文档写权限的经办人可确认发送

系统记录：

- 写入 `document_send_records`
- 更新 `sent_at`
- 更新执行状态为 `sent`

## 8. 归档规则

“对外发文”审批通过并由发起人确认发布后：

- 归档目标分类：`department`
- 归档目标目录：`codocs/departments/{dept_code}/outsides`
- 原文档 `publish_info.label`：`部门文档/对外发文`
- 一般文件支持下载为 `Markdown`、`DOCX`、`PDF`
- 重要文件和关键文件仅支持导出 `PDF`

因此，“对外发文”不再归档到公司级目录，而是归档到对应部门自己的 `outsides` 目录。

与之对应，组织资产栏目仅支持在线查看，不提供导出能力。

## 9. 数据字段约定

提交“对外发文”时，`document_reviews.extra` 中记录以下信息：

- `sendTo`
- `sendReason`
- `needsOfficialSeal`
- `outsideFileLevel`
- `businessDeptCode`
- `businessDeptName`
- `committeeDeptCode`
- `committeeDeptName`
- `sourceDeptCode`
- `sourceDeptName`
- `upperLeaderId`

这些字段用于：

- 审阅详情展示
- 审批流推进
- 一般文件审批完成后的上级领导通知

此外，发布后执行阶段还使用以下字段和记录：

- `document_reviews.execution_status`
- `document_reviews.published_document_uuid`
- `document_reviews.sealed_at`
- `document_reviews.sent_at`
- `document_seal_records`
- `document_send_records`

---

## 10. 展示与只读规则

- 已发布的对外发文统一只读，所有协同参与人均不可直接编辑正文
- 对外发文相关参与人可在协同文档中心查看全文
- 盖章记录、发送记录可在发布记录弹窗和审阅详情页查看
- 重要文件和关键文件仅支持导出 PDF；一般文件可导出 Markdown、DOCX、PDF

---

## 11. 当前实现边界

当前规则只覆盖：

- 部门文档中的“对外发文”
- 基于 `dept_category` 的资格判定
- 基于业务部门、委员会、文件级别的动态审批链
- 发布后的确认盖章、确认发送与执行留痕

不包含：

- 项目文档的对外发文扩展规则
- 委员会独立发文规则重构
- 审批模板对本规则的覆盖能力

“对外发文”当前采用固定业务规则生成审批链，不依赖后台审批模板配置。
