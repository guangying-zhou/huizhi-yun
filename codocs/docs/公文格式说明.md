# 公文格式说明

基于代码实现，本文档详细说明 Markdown 转 Word 工具的公文格式规范。

---

## 一、纸张设置

### 1.1 纸张规格
- **纸张型号**：A4
- **宽度**：11906 缇（约 21.0 厘米）
- **高度**：16838 缇（约 29.7 厘米）
- **页边距**：1440 缇（约 2.54 厘米/1 英寸）

### 1.2 单位说明
- **缇（Twip）**：1/1440 英寸，约等于 1/20 磅
- **磅（Point）**：1/72 英寸

## 三、标题格式

### 3.1 标题级别映射

Markdown 标题标记与公文标题级别的对应关系：

| Markdown      | 公文标题级别 | 字体           | 字号 | 序号格式                   | 加粗 | 缩进       |
| ------------- | ------------ | -------------- | ---- | -------------------------- | ---- | ---------- |
| `# 标题`      | 文档主标题   | 方正小标宋简体 | 二号 | 无                         | 是   | 无（居中） |
| `## 标题`     | 一级标题     | 黑体           | 三号 | `一、` `二、` `三、`       | 是   | 2 字符     |
| `### 标题`    | 二级标题     | 楷体_GB2312    | 三号 | `（一）` `（二）` `（三）` | 是   | 2 字符     |
| `#### 标题`   | 三级标题     | 仿宋_GB2312    | 三号 | `1.` `2.` `3.`             | 是   | 2 字符     |
| `##### 标题`  | 四级标题     | 仿宋_GB2312    | 三号 | `（1）` `（2）` `（3）`    | 否   | 2 字符     |
| `###### 标题` | 五级标题     | 仿宋_GB2312    | 三号 | `①` `②` `③`                | 否   | 2 字符     |

### 3.2 主标题处理规则

#### 情况一：Markdown 文件包含 `# 主标题`
- 优先使用 Markdown 中的 `# 标题` 作为文档主标题
- 居中对齐
- 段前/段后间距：400 缇
- 字体：方正小标宋简体，二号，加粗

#### 情况二：Markdown 文件无 `# 主标题`
- 自动使用上传文件的文件名作为主标题
- 去除文件扩展名
- 格式同上（方正小标宋简体、二号、加粗、居中）

### 3.3 标题序号自动生成

系统会自动为各级标题添加符合公文规范的序号：

#### 序号格式规则
1. **一级标题**：中文数字加顿号 `一、` `二、` `十、` `十一、` `二十、` ...
2. **二级标题**：括号加中文数字 `（一）` `（二）` `（十）` `（二十）` ...
3. **三级标题**：阿拉伯数字加点 `1.` `2.` `10.` `100.` ...
4. **四级标题**：括号加阿拉伯数字 `（1）` `（2）` `（10）` `（100）` ...
5. **五级标题**：圆圈数字 `①` `②` `③` `⑩` ...（超过10用阿拉伯数字）

#### 序号计数规则
- 每个级别的标题独立计数
- 下级标题出现时，更低级别的计数器重置为 0
- 自动去除原文本中已存在的序号，避免重复

### 3.4 标题间距与缩进

| 标题类型 | 段前间距 | 段后间距 | 首行缩进        | 对齐方式 |
| -------- | -------- | -------- | --------------- | -------- |
| 主标题   | 400 缇   | 400 缇   | 无              | 居中     |
| 其他标题 | 100 缇   | 200 缇   | 480 缇（2字符） | 左对齐   |

## 四、正文格式

### 4.1 段落设置

| 属性     | 值              | 代码实现                    |
| -------- | --------------- | --------------------------- |
| 字体     | 仿宋_GB2312     | `FONT_FANGSONG`             |
| 字号     | 三号（16磅）    | `FONT_SIZE_THREE`           |
| 行距     | 单倍行距        | `LINE_SPACING_SINGLE = 1.0` |
| 段前间距 | 0 缇            | `setSpacingBefore(0)`       |
| 段后间距 | 100 缇          | `setSpacingAfter(100)`      |
| 首行缩进 | 480 缇（2字符） | `INDENT_TWO_CHARS`          |
| 对齐方式 | 左对齐          | `ParagraphAlignment.LEFT`   |

### 4.2 特殊段落处理

#### 列表项
- 字体：仿宋_GB2312，三号
- 段前间距：50 缇
- 段后间距：100 缇
- 首行缩进：480 缇（2字符）

#### 引用块
- 字体：仿宋_GB2312，三号，**斜体**
- 段前间距：50 缇
- 段后间距：100 缇
- 左侧缩进：480 缇（2字符）

## 六、图片格式

### 6.1 图片设置

| 属性     | 值                |
| -------- | ----------------- |
| 默认宽度 | 400 像素          |
| 默认高度 | 300 像素          |
| 单位转换 | 1 像素 = 9525 EMU |
| 对齐方式 | 居中对齐          |

### 6.2 图片标题

- 字体：仿宋_GB2312，三号，加粗
- 对齐方式：居中对齐
- 段后间距：200 缇

### 6.3 特殊图片处理

#### 网络图片
- 自动下载并插入
- 支持 HTTP/HTTPS 协议

#### 本地图片
- 显示占位符：`[图片: 文件名]`
- 提示手动插入

#### 流程图（Flowchart/Mermaid）
- 显示占位符：`[流程图: 标题]`
- 提示建议使用 Visio 或 Draw.io 编辑后手动粘贴

## 七、间距与缩进

### 7.1 间距设置汇总

| 元素类型     | 段前间距（缇） | 段后间距（缇） |
| ------------ | -------------- | -------------- |
| 主标题       | 400            | 400            |
| 其他标题     | 100            | 200            |
| 普通段落     | 0              | 100            |
| 列表项       | 50             | 100            |
| 引用块       | 50             | 100            |
| 图片         | 200            | 200            |
| 流程图占位符 | 300            | 300            |
| 图片说明     | 0              | 200            |

### 7.2 缩进设置汇总

| 元素类型         | 左侧缩进（缇） | 首行缩进（缇） | 说明           |
| ---------------- | -------------- | -------------- | -------------- |
| 普通段落         | -              | 480            | 2 字符         |
| 标题（除主标题） | -              | 480            | 2 字符         |
| 列表项           | -              | 480            | 2 字符         |
| 引用块           | 480            | -              | 2 字符左侧缩进 |

### 7.3 缩进计算

- **基准字体**：三号字（16磅）
- **字符宽度**：约 240 缇/字符
- **2 字符缩进**：480 缇（`INDENT_TWO_CHARS` 常量）

## 八、字体兼容性要求

### 8.1 必需字体

为确保转换效果，系统需安装以下字体：

| 字体名称       | 用途           | 来源             |
| -------------- | -------------- | ---------------- |
| 黑体           | 一级标题       | Windows 系统自带 |
| 楷体_GB2312    | 二级标题       | Windows 系统自带 |
| 仿宋_GB2312    | 正文、其他标题 | Windows 系统自带 |
| 方正小标宋简体 | 文档主标题     | 需单独安装       |

### 8.2 字体说明

- 以上字体均符合《党政机关公文格式》（GB/T 9704-2012）要求
- 如字体缺失，Word 会使用替代字体，可能导致格式不符合公文规范
- 建议使用 Windows 系统环境，以确保字体完整

## 九、使用示例

### 9.1 Markdown 示例

```markdown
# 关于加强信息化建设工作的通知

各相关部门：

为进一步推进信息化建设，提高工作效率，现将有关事项通知如下：

## 总体要求

（一）加强组织领导
1. 成立信息化建设工作小组
2. 明确各部门职责分工
3. 定期召开工作例会

（二）完善制度建设
- 制定信息化管理办法
- 建立安全管理制度
- 规范数据管理流程

## 具体措施

1. 技术升级
（1）更新硬件设备
（2）升级软件系统
① 服务器升级
② 数据库优化

2. 人员培训
（1）开展技术培训
（2）组织考核评估

## 工作要求

请各部门高度重视，确保按时完成任务。

特此通知。

信息化工作小组
2024年1月1日
```

### 9.2 转换后 Word 格式

- **主标题**：关于加强信息化建设工作的通知（方正小标宋简体，二号，加粗，居中）
- **一级标题**：一、总体要求（黑体，三号，加粗，左对齐，2字符缩进）
- **二级标题**：（一）加强组织领导（楷体_GB2312，三号，加粗，左对齐，2字符缩进）
- **三级标题**：1. 技术升级（仿宋_GB2312，三号，加粗，左对齐，2字符缩进）
- **四级标题**：（1）更新硬件设备（仿宋_GB2312，三号，左对齐，2字符缩进）
- **五级标题**：① 服务器升级（仿宋_GB2312，三号，左对齐，2字符缩进）
- **正文**：仿宋_GB2312，三号，左对齐，2字符首行缩进，单倍行距

## 十、注意事项

1. **标题层级**：Markdown 中使用 `#` 至 `######` 分别对应文档主标题至五级标题
2. **序号自动生成**：系统会自动删除原有序号并重新生成，避免重复
3. **空行处理**：空段落会被自动忽略
4. **图片限制**：本地图片暂不支持自动插入，需手动添加
5. **流程图支持**：仅显示占位符，建议使用专业工具编辑
6. **字体要求**：确保系统已安装所需中文字体
7. **文件大小**：上传文件限制为 10MB
8. **支持格式**：仅支持 `.md` 和 `.markdown` 格式文件

## 十一、技术实现要点

### 11.1 核心技术栈

- **Apache POI 5.2.5**：生成 Word 文档
- **CommonMark 0.21.0**：解析 Markdown 文件
- **TablesExtension**：支持 Markdown 表格解析

### 11.2 关键常量定义

```java
// 字体定义
FONT_SIMHEI = "黑体"
FONT_KAITI = "楷体_GB2312"
FONT_FANGSONG = "仿宋_GB2312"
FONT_FANGZHENG_XIAOBIAOSONG = "方正小标宋简体"

// 字号定义（单位：磅）
FONT_SIZE_THREE = "16"  // 三号字
FONT_SIZE_TWO = "22"    // 二号字

// 间距与缩进（单位：缇）
LINE_SPACING_SINGLE = 1.0      // 单倍行距
INDENT_TWO_CHARS = 480         // 2字符缩进
MARGIN = 1440                  // 页边距

// 纸张规格（单位：缇）
PAGE_WIDTH_A4 = 11906          // A4宽度
PAGE_HEIGHT_A4 = 16838         // A4高度
```

### 11.3 中文数字转换

支持 1-20 的中文数字自动转换，用于一级和二级标题序号：
- 1-10：一至十
- 11-19：十一至十九
- 20+：二十、二十一、三十...

## 附录：公文格式标准

本工具遵循以下国家标准：

- **GB/T 9704-2012**《党政机关公文格式》
- **GB/T 9704-1988**《国家机关公文格式》

所有格式参数均按照标准要求设置，确保生成文档符合公文排版规范。
