## 2. 产品概述

Altoc 是一款面向 ToB / ToG 项目型企业的 AI 辅助 LTC（Lead to Cash）经营平台，聚焦线索、客户、商机、报价、合同、交付、验收、回款、续约的全链路管理。

Altoc 的核心目标不是替代 ERP、OA 或专业财务系统，而是围绕经营结果打通销售、交付与回款之间的断点，让管理层看得清、销售团队推得动、项目经营收得回。

核心经营链路：

**线索 -> 客户 -> 商机 -> 报价 -> 合同 -> 交付 -> 验收 -> 回款 -> 续约 / 运维**

## 3. 产品背景

项目型企业尤其是 IT 服务商、系统集成商、政企解决方案公司通常存在以下问题：

* 客户和商机数据分散在表格、聊天和个人记录中，销售过程无法透明管理
* 销售只重签约，不重报价纪律、毛利控制和回款闭环
* 报价、合同、交付、验收、回款分属不同系统，信息断裂严重
* ToG 项目招投标流程长，关键节点多，材料与进度协同成本高
* 老板难以获得可靠预测，财务难以及时掌握应收风险
* 存量客户续约、维保、增购缺乏系统经营机制

## 4. 产品目标

### 4.1 业务目标

* 建立覆盖销售到回款的统一经营主线
* 提升销售过程透明度和预测可信度
* 规范报价、审批、合同、回款相关管理规则
* 打通商机、交付、验收、回款的数据关系
* 为 ToB / ToG 项目经营提供统一视图与经营分析

### 4.2 MVP 目标

MVP 一期优先解决三个问题：

* 管理层看清 pipeline、合同额、回款额和应收风险
* 销售团队把客户、商机、报价、合同、回款串成同一条流程
* 围绕关键节点建立基础规则和预警机制，而不是只做数据记录

### 4.3 非目标

以下内容不在 MVP 范围内：

* 完整财务账套
* 完整进销存体系
* 通用 OA 能力
* 面向所有行业的深度行业定制
* 大规模营销自动化平台

## 5. 目标客户与适用场景

### 5.1 目标客户画像

* ToB 企业：系统集成商、软件开发商、实施服务商、SaaS 方案销售团队
* ToG 企业：面向政府、事业单位、国企开展项目型销售、招投标、交付和维保的服务型企业
* 企业规模：中小企业为主，兼顾成长型团队
* 销售特征：客单价较高、销售周期较长、多人协同、流程复杂、回款依赖合同和验收节点

### 5.2 核心行业

* 政务信息化
* 企业数字化服务
* 软件与解决方案销售
* 集成与运维服务

### 5.3 典型业务场景

#### 场景 A：ToB 解决方案销售

销售跟进客户需求，沉淀联系人和组织关系，推进商机，生成报价并审批，签约后联动交付与回款，后续再经营续约和增购。

#### 场景 B：ToG 招投标项目

团队围绕政府客户推进项目线索、立项信息、招标进度、投标协同、合同签订、实施验收和回款计划，重点管理关键角色、投标节点、验收材料和质保金。

#### 场景 C：存量客户续约与维保

围绕维保合同、SLA、服务工单和客户满意度持续经营客户，识别续约与交叉销售机会，提升客户生命周期价值。

## 6. 核心角色

### 6.1 用户角色

* 销售人员：录入客户、维护商机、提交报价、推进合同、登记回款进度
* 销售经理：配置阶段规则、审核预测、监督商机质量和团队达成
* 售前 / 方案经理：支持方案、投标、竞品和资料协同
* 交付负责人：跟踪合同执行、项目里程碑和验收节点
* 财务 / 经营人员：管理开票、到账、核销和应收风险
* 企业负责人：查看整体经营数据、风险和预测
* 系统管理员：维护基础配置、审批规则、权限和集成参数

### 6.2 角色关注点

* 销售人员关注：是否好录入、好跟进、好推进
* 销售经理关注：是否能判断商机真假、预测准不准、团队动作是否到位
* 财务关注：是否知道该开票什么、何时回款、哪些应收逾期
* 负责人关注：收入机会、利润风险、回款节奏、续约机会

## 8. 产品原则

* 围绕经营闭环设计，而非围绕部门孤岛设计
* 优先支持项目型组织的销售和回款管控
* 规则优先于记录，流程优先于表单
* 默认提供行业通用模板，同时允许客户做轻量配置
* AI 必须服务具体业务动作，不做空泛展示

## 9. 业务流程

### 9.1 主流程

1. 录入线索
2. 线索分配与跟进
3. 线索转客户 / 商机
4. 商机推进与销售活动沉淀
5. 创建报价并提交审批
6. 报价发送客户并迭代版本
7. 商机赢单后创建合同
8. 合同审批通过后生效
9. 根据合同付款条款自动生成回款计划
10. 跟踪交付、验收、开票与到账
11. 回款完成后进入维保 / 续约 / 增购经营

### 9.2 ToG 扩展流程

1. 跟踪立项信息
2. 维护预算、招标、投标关键节点
3. 关联方案、资质、案例等投标材料
4. 中标后转入合同和交付流程
5. 按验收节点、结算节点和质保节点管理回款

### 10.1 核心对象

* 线索
* 客户
* 联系人
* 商机
* 销售活动
* 报价单
* 合同
* 合同付款条款
* 回款计划
* 发票
* 回款记录
* 项目 / 交付里程碑
* 运维 / 续约机会

### 10.2 主要关系

* 一个客户可关联多个联系人
* 一个客户可关联多个商机
* 一个商机可关联多次销售活动和多个报价版本
* 一个赢单商机通常对应一个主合同，也可扩展多个补充合同
* 一个合同可拆分多个付款节点，自动形成回款计划
* 一个合同可关联一个或多个项目 / 里程碑
* 一个客户可产生维保、续约、增购等后续机会

## 11. 功能需求

### 11.1 客户与联系人管理

#### 目标

建立客户主数据和关键人关系视图，为商机推进、投标、合同和回款提供基础信息。

#### 功能点

* 新建、编辑、归档客户
* 客户类型区分：企业、政府、事业单位、国企、渠道伙伴
* 联系人管理：姓名、部门、岗位、电话、邮箱、角色标签
* 客户标签体系：行业、区域、等级、来源、状态
* 客户关系地图：决策人、使用人、采购人、技术影响人
* 客户跟进记录与提醒
* 客户去重与合并

#### 关键字段

客户：

* 客户名称
* 客户类型
* 所属行业
* 区域
* 客户等级
* 所有者
* 来源渠道
* 状态

联系人：

* 姓名
* 所属客户
* 部门
* 岗位
* 联系方式
* 决策角色
* 是否关键人

#### 业务规则

* 同一组织名称不得重复创建有效客户，允许管理员合并
* 客户必须指定归属人
* 商机、报价、合同必须绑定客户
* 关键联系人缺失时，商机进入高风险提示

#### 验收要点

* 销售可在 3 分钟内完成客户和首个联系人的录入
* 支持按行业、区域、等级、归属人快速检索客户
* 客户详情页可看到联系人、商机、合同、回款摘要

### 11.2 线索管理

#### 目标

承接销售入口信息，形成可分配、可跟进、可转化的线索池。

#### 功能点

* 线索录入：手工录入为主，后续支持导入和接口接入
* 线索来源管理：市场活动、渠道推荐、主动拜访、老客户转介绍、招标信息
* 线索分配与回收
* 线索跟进记录
* 线索转客户 / 商机
* 无效线索关闭与原因沉淀
* AI 线索评分

#### 状态定义

* 新建
* 待分配
* 跟进中
* 已转商机
* 已关闭无效

#### 业务规则

* 线索可转为客户和商机，保留转化链路
* 已关闭无效线索需填写关闭原因
* 连续 N 天未跟进的线索触发提醒，N 默认 7 天

### 11.3 商机管理

#### 目标

以商机为经营中心，管理推进质量、预测口径和赢单概率。

#### 功能点

* 新建商机
* 商机阶段流转
* 商机金额、预计签约时间、预计回款时间管理
* 商机风险、竞品、关键人、下一步动作记录
* 商机预测分类管理
* 商机赢单 / 输单 / 暂停
* 商机分析：赢率、周期、转化率、风险分布

#### 默认阶段

* 初步接触
* 需求确认
* 方案沟通
* 报价 / 投标
* 商务谈判
* 待签约
* 赢单
* 输单
* 暂停

#### 必填字段

* 商机名称
* 所属客户
* 预计金额
* 当前阶段
* 预计签约日期
* 商机负责人

#### 业务规则

* 阶段流转可配置，每阶段支持退出标准和必填项
* 进入“报价 / 投标”前必须填写预计金额和主要竞争对手
* 进入“待签约”前必须存在已审批通过的报价或投标结果
* 赢单后可一键发起合同
* 输单必须填写输单原因和主要丢单对手

#### AI 支持

* 根据商机进展、活动频次、关键人完整度生成风险提示
* 根据历史同类商机给出赢率和下一步动作建议

### 11.4 销售活动与过程管理

#### 目标

沉淀销售行为过程，让商机推进有证据、有动作、有责任人。

#### 功能点

* 记录拜访、电话、演示、会议、招投标、纪要
* 创建任务和提醒
* 关联客户、联系人、商机
* 记录下一步动作、承诺时间和责任人
* 销售预测分类：Pipeline、Best Case、Commit

#### 业务规则

* 商机必须存在下一步动作和预计完成时间
* 超过 N 天无活动的商机进入“停滞风险”，N 默认 14 天
* Commit 商机必须满足阶段、金额、关键人和报价状态的规则校验

### 11.5 报价管理

#### 目标

规范报价版本、审批和毛利边界，为合同签订提供可信输入。

#### 功能点

* 维护产品 / 服务目录
* 新建报价单
* 报价明细、折扣、税率、毛利展示
* 报价版本管理
* 报价审批流对接 Workflow
* 报价导出与发送记录
* 报价接受 / 失效 / 作废管理

#### 报价状态

* 草稿
* 待审批
* 审批通过
* 审批拒绝
* 已发送
* 已接受
* 已失效
* 已作废

#### 关键字段

* 报价编号
* 所属客户
* 所属商机
* 报价金额
* 折扣率
* 税率
* 毛利率
* 有效期
* 当前状态

#### 业务规则

* 报价必须关联客户和商机
* 折扣超阈值时必须进入审批
* 毛利率低于红线时必须升级审批
* 仅审批通过状态可发送客户
* 已接受报价可一键转合同

### 11.6 合同管理

#### 目标

基于赢单商机形成受控合同，并为交付和回款提供经营依据。

#### 功能点

* 合同创建、编辑、审批
* 合同模板管理
* 合同金额、开始结束日期、付款条款管理
* 补充协议和变更管理
* 合同到期提醒
* 合同执行状态跟踪

#### 合同状态

* 草稿
* 审批中
* 已生效
* 执行中
* 已完成
* 已终止
* 已到期

#### 关键字段

* 合同编号
* 合同名称
* 所属客户
* 关联商机
* 合同金额
* 签约日期
* 生效日期
* 结束日期
* 付款条款
* 质保金比例
* 当前状态

#### 业务规则

* 合同必须关联客户
* 合同金额默认继承已接受报价，可允许人工调整并保留日志
* 合同生效后自动生成回款计划
* 合同变更必须保留版本历史
* 到期前 N 天自动预警，N 默认 30 天

### 11.7 回款与应收管理

#### 目标

围绕合同付款条款管理计划、开票、到账和逾期应收，形成回款闭环。

#### 功能点

* 根据合同付款条款生成回款计划
* 回款节点支持类型：预付款、里程碑款、验收款、质保金
* 开票申请与发票记录
* 回款登记与核销
* 逾期提醒和风险分级
* 按客户、合同、负责人查看回款进度

#### 回款计划状态

* 待开始
* 待开票
* 待回款
* 部分回款
* 已回款
* 已逾期
* 坏账

#### 关键字段

* 关联合同
* 节点名称
* 节点类型
* 计划金额
* 计划回款日期
* 计划开票日期
* 已回款金额
* 未回款金额
* 当前状态

#### 业务规则

* 合同生效后必须存在至少一个回款计划节点
* 已回款金额不可超过节点计划金额，超额时需确认是否预收
* 到期未回款自动标记逾期
* 逾期天数达到阈值时提升风险等级
* 应收看板默认展示未来 30 天到期和已逾期数据

### 11.8 经营看板与分析

#### 目标

为管理层提供统一、可追溯的经营数据视图。

#### 功能点

* 销售漏斗看板
* 商机预测看板
* 合同金额趋势
* 回款趋势和应收预警
* 销售人员目标达成情况
* 客户和行业维度分析

#### 核心指标

* 线索转商机率
* 商机赢单率
* 平均销售周期
* 报价转合同率
* 合同金额
* 毛利率
* 回款达成率
* 逾期应收金额
* DSO
* 续约率
* 销售预测准确率

### 11.9 系统设置与权限

#### 目标

支持基础主数据、流程模板和权限模型配置。

#### 功能点

* 行业、区域、客户等级、客户类型配置
* 销售阶段模板配置
* 报价审批规则配置
* 付款条款模板配置
* 角色权限设置
* 数据权限设置

#### 默认权限模型

* 销售人员：查看本人及授权客户 / 商机数据
* 销售经理：查看所属团队数据，配置阶段与预测规则
* 财务人员：查看合同、开票、回款与应收数据
* 管理层：查看全局分析和经营数据
* 管理员：管理配置、权限和集成

#### 数据权限原则

* 客户、商机、合同数据默认按归属组织和归属人控制
* 金额、毛利、回款数据可单独配置可见范围
* 所有关键状态变更需保留操作日志

### 11.10 AI 能力

#### MVP 范围

* AI 客户摘要
* AI 商机风险提示
* AI 销售纪要结构化

#### 输入来源

* 客户档案
* 联系人信息
* 销售活动记录
* 商机阶段与金额
* 报价与合同摘要

#### 输出形式

* 摘要卡片
* 风险标签
* 下一步动作建议
* 纪要结构化字段回填建议

#### 业务规则

* AI 输出仅作为建议，不直接修改业务字段
* 所有 AI 生成结果需允许人工确认
* 对 AI 推荐的采纳与否可记录，用于后续优化

## 12. 关键状态机建议

### 12.1 线索状态机

* 新建 -> 待分配 -> 跟进中 -> 已转商机
* 新建 / 待分配 / 跟进中 -> 已关闭无效

### 12.2 商机状态机

* 初步接触 -> 需求确认 -> 方案沟通 -> 报价 / 投标 -> 商务谈判 -> 待签约 -> 赢单
* 任意前置阶段 -> 输单
* 任意前置阶段 -> 暂停

### 12.3 报价状态机

* 草稿 -> 待审批 -> 审批通过 -> 已发送 -> 已接受
* 草稿 -> 待审批 -> 审批拒绝
* 已发送 -> 已失效
* 草稿 / 审批通过 / 已发送 -> 已作废

### 12.4 合同状态机

* 草稿 -> 审批中 -> 已生效 -> 执行中 -> 已完成
* 已生效 / 执行中 -> 已终止
* 已生效 / 执行中 -> 已到期

### 12.5 回款计划状态机

* 待开始 -> 待开票 -> 待回款 -> 部分回款 -> 已回款
* 待回款 / 部分回款 -> 已逾期
* 已逾期 -> 坏账

## 13. 集成需求

### 13.1 Workflow

* 报价审批
* 合同审批
* 费用报销审批

### 13.2 Align

* 售前协作任务
* 客户拜访安排
* 商机作战室

### 13.3 Codocs

* 投标材料关联
* 方案文档关联
* 运维知识库关联

### 13.4 Aims

* 合同与项目里程碑关联
* 交付节点、验收节点回传
* 工时和交付状态同步

### 13.5 财务系统

* 同步发票和到账结果
* 对接金蝶、用友等系统
* 保持 Altoc 为经营前台，财务系统为账务后台

## 14. 非功能需求

### 14.1 可用性

* 关键业务表单应支持保存草稿
* 列表页支持筛选、排序、导出
* 页面默认面向中文业务用户设计

### 14.2 安全与审计

* 支持角色权限与数据权限控制
* 关键字段变更必须保留日志
* 审批、状态流转、金额变更需可审计

### 14.3 性能

* 常用列表页在标准分页下应在可接受时间内加载完成
* 看板统计支持按日缓存或预聚合扩展

### 14.4 可配置性

* 销售阶段、客户等级、付款条款、审批规则可配置
* 不同客户组织可有不同销售流程模板

## 15. 报表与数据口径

### 15.1 统一口径要求

* 商机金额以主币种税前或税后口径统一配置
* 合同金额口径需明确含税 / 不含税
* 回款达成率按计划期内应收与实收计算
* 销售预测准确率按预测窗口和最终结果对比

### 15.2 必备报表

* 销售漏斗报表
* 商机预测报表
* 合同额趋势报表
* 应收账龄报表
* 回款计划执行报表
* 销售人员业绩报表
* 客户贡献与行业分布报表

## 16. MVP 验收标准

### 16.1 业务验收

* 销售可完成客户、商机、报价、合同、回款计划的完整串联
* 管理层可查看团队 pipeline、commit 和回款风险
* 财务可查看未来 30 天回款计划和逾期应收
* 商机、报价、合同关键状态变更有日志可追溯

### 16.2 产品验收

* 所有 P0 模块均具备基本可用闭环
* 默认流程可跑通，关键状态可流转
* 关键字段校验与规则提醒生效
* 基础权限模型可工作

### 16.3 AI 验收

* 客户摘要可基于现有客户和活动信息生成
* 商机风险提示可识别停滞、关键人缺失、无下一步动作等问题
* 销售纪要结构化可将自由文本转成至少客户、需求、风险、下一步四类信息

## 17. 风险与待确认事项

### 17.1 风险

* 目标客户同时覆盖 ToB / ToG，首版流程配置需避免过重
* 若审批、文档、项目系统接口准备不足，部分联动能力会受阻
* 若商机阶段规则定义不清，预测与分析会失真
* 若主数据质量不足，AI 输出价值有限

### 17.2 待确认

* 首版是否需要渠道伙伴报备与冲突管理
* ToG 场景是否在 MVP 即纳入招投标模块，还是先以商机扩展字段承接
* 财务系统对接是单向同步还是双向回写
* 是否需要多组织、多公司主体支持
* 金额、税率、毛利的默认口径如何定义

## 18. 里程碑建议

完成客户、联系人、线索、商机和销售活动基础闭环。
