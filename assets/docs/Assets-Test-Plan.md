# Assets MVP Test Plan

状态：DRAFT  
日期：2026-03-21  
范围：Assets MVP 当前已交付功能的统一测试与验收口径

## 1. 目标

本测试计划用于验证 Assets MVP 是否已经完成以下闭环：

- 权限、菜单、页面路由可正常访问
- 首页、列表页、详情页能基于 MySQL 真实数据正常工作
- 主要录入与维护动作均通过模态窗口完成
- 采购、入库、预警等关键业务操作能形成最小闭环
- 系统字典维护后，业务表单下拉项可同步反映变更

## 2. 测试前提

- 已启动 `assets` 应用
- 已完成 `assets/docs/assets_schema.sql` 建表
- 已执行 `assets/docs/assets_seed.sql` 联调种子数据
- 当前登录账号已具备 Assets 对应资源权限
- `Account` 认证与权限接口可正常返回

## 3. 范围内功能

本轮统一测试覆盖：

- 权限加载与菜单显示
- 工作台
- 实物资产列表、详情、新增、编辑
- 资源资产列表、详情、新增、编辑
- 产品资产列表、详情、新增、编辑、关联技术底座、关联资源、关联文档
- 知识产权资产列表、详情、新增、编辑、关联产品、关联文档
- 数字资产列表、详情、新增、编辑、关联产品、关联文档
- 环境列表、详情、新增、编辑
- 客户交付列表、详情编辑、关联产品
- 客户交付关联环境
- 供应商列表、新增、编辑
- 采购单列表、详情、新增、编辑、采购明细维护、提交审批
- 入库/激活记录新增
- 预警列表、筛选、处理
- 报表页
- 系统管理中的字典管理

本轮不作为阻塞项的内容：

- 批量导出
- Workflow 真审批集成
- Account / Altoc / Aims / Codocs 的真实选择器与联动
- 大数据量性能压测
- 全量 lint 遗留清理

## 4. 验收场景

### 4.1 权限与导航

1. 登录后打开首页 `/`
2. 验证左侧菜单可见，且包含工作台、资产台账、环境视图、客户交付、采购管理、预警中心、报表统计、系统管理
3. 刷新页面，确认不再出现未登录状态下的 `/api/auth/permissions` 401 噪音
4. 点击主要菜单，确认能正常跳转到对应页面

### 4.2 工作台

1. 打开 `/`
2. 验证摘要卡片、待处理预警、即将到期资产可正常展示
3. 点击预警或资产入口，确认能跳转到对应模块页面
4. 点击刷新，确认页面能重新拉取数据

### 4.3 实物资产

1. 打开 `/assets/physical`
2. 验证搜索与状态筛选会触发服务端查询，列表结果变化正确
3. 点击“新增资产”，通过模态窗口录入一条实物资产
4. 创建成功后确认跳转到详情页 `/assets/[id]`
5. 在详情页点击“编辑”，修改名称、负责人或备注并保存
6. 返回列表，确认修改结果已生效

### 4.4 资源资产

1. 打开 `/assets/resources`
2. 验证搜索与状态筛选会触发服务端查询，列表结果变化正确
3. 点击“新增资源”，通过模态窗口录入一条资源资产
4. 填写月度成本、到期时间、资源类型等信息并保存
5. 进入详情页后再编辑一次，确认状态、备注、到期信息可更新

### 4.5 环境视图

1. 打开 `/environments`
2. 验证搜索与状态筛选会触发服务端查询
3. 点击“新增环境”，通过模态窗口创建一条环境记录
4. 创建成功后确认跳转至 `/environments/[id]`
5. 在详情页编辑维护人、拓扑摘要、备注，确认保存成功

### 4.6 产品资产

1. 打开 `/assets/products`
2. 验证列表可同时看到产品主档和技术底座，并能按子类别区分
3. 点击“新增资产”，分别创建一条产品主档和一条技术底座
4. 进入 `/products/[id]`，编辑产品信息并关联技术底座、资源资产、文档
5. 进入 `/technology-bases/[id]`，编辑技术底座并确认关联产品展示正确

### 4.7 知识产权资产

1. 打开 `/assets/ip`
2. 验证搜索与状态筛选可正常工作
3. 通过模态窗口新建一条知识产权资产
4. 进入 `/ip-assets/[id]` 后编辑权属信息
5. 关联产品和文档，确认详情页列表刷新后可见

### 4.8 数字资产

1. 打开 `/assets/digital`
2. 验证搜索与状态筛选可正常工作
3. 通过模态窗口新建一条数字资产
4. 进入 `/digital-assets/[id]` 后编辑存储位置、访问权限、状态
5. 关联产品和文档，确认详情页列表刷新后可见

### 4.9 客户交付

1. 打开 `/deliveries`
2. 进入任一详情页 `/deliveries/[id]`
3. 点击“编辑”，修改状态、计划上线时间、验收时间或备注
4. 关联产品，确认详情页产品列表已更新
5. 关联环境，确认详情页环境列表和环境数量更新
6. 保存后确认详情页展示已更新

### 4.10 供应商

1. 打开 `/procurement/suppliers`
2. 点击“新增供应商”，通过模态窗口创建记录
3. 在列表中点击该供应商，再次通过模态窗口编辑
4. 确认类型、状态、联系人、发票信息等字段更新成功

### 4.11 采购单与入库

1. 打开 `/procurement/orders`
2. 验证搜索与状态筛选会触发服务端查询
3. 点击“新增采购单”，通过模态窗口创建采购单
4. 创建成功后跳转到 `/procurement/orders/[id]`
5. 在详情页点击“编辑”，修改采购类型、用途、预算或备注
6. 点击“新增明细”，录入一条采购项；再点明细行，修改数量、金额或目标归属
7. 确认预算金额会随明细总价变化刷新
8. 点击“提交审批”，确认状态变更为待审批或已提交后的目标状态
9. 点击“新增入库记录”，录入一条入库/激活记录
10. 刷新详情页，确认入库记录已显示

### 4.12 预警中心

1. 打开 `/alerts`
2. 验证搜索与状态筛选会触发服务端查询
3. 点击任一预警行，打开处理模态
4. 修改状态、处理说明、下次提醒时间并保存
5. 返回列表确认状态已变化

### 4.13 报表统计

1. 打开 `/reports`
2. 确认总览、到期、项目成本、客户交付成本等卡片/列表正常展示
3. 刷新页面，确认无接口报错

### 4.14 字典管理

1. 打开 `/admin/dictionaries`
2. 选择任一字典并编辑选项文案
3. 保存成功后，重新打开对应业务表单
4. 验证相关字段已经以下拉框方式展示，并反映字典最新值

## 5. 重点检查项

- 主要录入和主要信息维护必须使用模态窗口，不应退回抽屉式交互
- 所有列表页搜索和状态筛选应由服务端处理，不应只做前端本地筛选
- 详情页保存后，列表页和详情页展示要保持一致
- 权限加载必须是 fail-closed，未登录不应误放行
- 字典变更后，业务表单下拉项要能刷新生效

## 6. 已知限制

- 当前列表筛选只覆盖搜索和单一状态，不包含更复杂的高级筛选条件
- 供应商目前以列表页内编辑为主，尚无独立详情页
- 审批流目前仍是 Assets 本地状态流转，未接入 Workflow 实例
- 部分字段仍依赖手工输入业务键，尚未接入跨模块选择器

## 7. 通过标准

满足以下条件即可判定当前 MVP 可进入下一阶段：

- 所有 P0 页面可访问且无阻塞性报错
- 所有 P0 创建、编辑、提交、处理动作可成功写入数据库
- 搜索、筛选、刷新、跳转行为正常
- 字典管理与业务表单下拉联动可验证
- 无新增 `typecheck` 错误
