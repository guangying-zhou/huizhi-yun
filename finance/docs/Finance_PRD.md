# Finance PRD

## 1. 文档信息

- 产品名称：Finance（汇智云财务）
- 文档名称：经营财务中台产品需求说明书
- 当前版本：v0.1 草案
- 创建日期：2026-05-15
- 适用范围：中小型项目制企业的经营财务、费用审批、项目核算、预算现金流、财务自动化与绩效金额财务口径

## 2. 产品定位

Finance 是汇智云平台中的经营财务中台，服务中小型项目制企业的真实资金流、费用流和经营核算。

它不是完整会计软件或 ERP 总账系统，首期不替代专业财务软件的总账、税务申报、多账套、复杂关账和银企直连。Finance 的核心价值是把合同、开票、到账、支出、核销、项目成本、预算现金流和绩效金额财务口径统一成可追溯的经营财务事实源，并把摘要反馈给 Altoc、Aims、Assets、People 和管理看板。

对于没有采购专业财务软件的中小企业，Finance 可以替代大量日常经营财务管理工作，例如合同台账、开票、回款、应收、费用、项目支出、预算、现金流预测和经营看板；但不直接承担法定总账、纳税申报和审计报表职责。后续版本可以提供凭证草稿、科目映射和标准导出能力，辅助对接金蝶、用友、畅捷通等专业财务软件。

## 3. 背景与依据

### 3.1 平台背景

汇智云目前已有以下业务边界：

- Altoc：经营前台，负责客户、商机、报价、合同、经营回款计划和应收风险。
- Aims：项目交付，负责项目、迭代、任务、里程碑和验收过程。
- Assets：资产与采购，负责资产台账、采购入库、分配、处置和成本归因。
- Workflow：通用审批流程引擎。
- Console/Integration：企业基础运行、目录、集成配置、凭证托管和应用入口。
- Platform：应用治理、授权、订阅和运行时策略。

Finance 要补齐资金事实源和经营财务核算，避免发票、到账、费用和核算逻辑分散在 Altoc、Aims、Assets 或外部表格中。

### 3.2 wizbizdb 存量依据

`wizbizdb` 中已有明显的财务能力：

- `wb_invoice`：发票台账。
- `wb_project_income`：项目收入/到账。
- `wb_project_payment`：项目支出。
- `wb_bank_account`：银行账户。
- `wb_account_balance`：账户余额快照。
- `wb_project_contribution`：项目成员贡献。
- `wb_salary`、`wb_attendance`、`wb_employee`：可为 People 人员事实、人力成本、个人绩效基础和 Finance 项目核算提供输入。

这些数据已经超过 Altoc 的 LTC 经营边界。继续放在 Altoc 会导致“经营预测”和“财务事实”混杂，后续支出、报销、核销、项目成本和绩效金额财务口径也会失去统一口径。

## 4. 产品目标

### 4.1 业务目标

- 建立统一的发票、到账、支出、银行账户和核销事实源。
- 支持费用报销、项目支出、付款申请等财务审批流程。
- 对接钉钉或企业微信，实现审批待办、审批结果同步和消息通知。
- 对接 People，支撑项目财务核算、人员成本分摊和绩效金额/提成奖金财务口径。
- 为 Altoc 提供合同开票、到账、核销摘要。
- 为 Aims 提供项目收入、支出、成本和毛利视图。
- 为企业负责人提供现金流、利润、费用和绩效口径的经营财务看板。

### 4.2 非目标

以下能力不进入 v0.1-v0.5 范围：

- 完整总账。
- 法定会计凭证自动制单。
- 税务申报。
- 多账套。
- 复杂会计期间关账。
- 固定资产折旧全流程。
- 工资个税社保核算。
- 银企直连和自动拉取银行流水。
- 替代金蝶、用友等专业财务软件。

Finance 后续可以导出数据或对接外部专业财务软件，但当前定位是经营财务中台。

## 6. 模块边界

### 6.1 Finance 负责

- 发票台账和开票申请落账。
- 到账记录、收款确认和收款核销。
- 非合同收入归类。
- 项目支出、费用报销、付款申请和支出台账。
- 银行账户和账户余额快照。
- 项目财务汇总：收入、支出、成本、毛利。
- 绩效金额财务口径：财务贡献归因、提成、奖金或绩效金额快照，供 People 个人绩效主流程引用。
- 应收、应付、预算、资金计划和经营财务看板。
- 银行流水导入、电子发票识别、凭证草稿、税务辅助和月结快照。
- 与 Workflow、钉钉、企业微信、People、Altoc、Aims 的财务侧集成。

### 6.2 Finance 不负责

- 客户、商机、报价、合同经营过程：由 Altoc 负责。
- 项目任务、里程碑和交付验收过程：由 Aims 负责。
- 员工、部门、岗位、职级、入离职事实、个人绩效周期/评分/确认/归档：由 People 或 Console Directory 负责。
- 审批流引擎：由 Workflow 负责。
- 钉钉/企业微信凭证托管：由 Console/Integration 负责。
- 资产采购、入库、领用、处置：由 Assets 负责，Finance 只承接付款、费用和成本结果。

## 7. 版本规划

### 7.1 v0.1：财务台账与核销

目标：先把真实财务事实源建起来，承接 OA 存量发票、收入、支出、银行账户数据。

范围：

- 发票台账：正式发票、开票金额、发票日期、合同关联、客户关联。
- 收款台账：到账日期、金额、付款方、渠道、银行账户、合同关联。
- 收款核销：到账记录核销到合同、发票或 Altoc 回款计划。
- 支出台账：项目支出、销售费用、采购、退款、银行手续费等。
- 银行账户：账户资料和余额快照。
- 非合同收入待归类：利息、补贴、借款、费用分摊等。
- 合同财务摘要 API：按 `contractCode` 返回开票、到账、核销和未收款摘要。
- OA 迁移支撑：导入 `wb_invoice`、`wb_project_income`、`wb_project_payment`、`wb_bank_account`、`wb_account_balance`。

验收标准：

- 能迁入 OA 财务核心台账，并保留 `legacy_source/legacy_id`。
- Finance 成为发票、到账、支出和银行账户事实源。
- Altoc 可通过 API 查询合同财务摘要。
- 非合同收入不进入 Altoc，统一进入 Finance 待归类池。

### 7.2 v0.2：报销、支出和付款审批

目标：把财务业务单据接入审批流程，并通过钉钉或企业微信触达用户。

范围：

- 费用报销单：员工报销、明细、票据附件、费用类型和成本归属。
- 项目支出申请：项目采购、外协、销售费用、项目费用、退款等。
- 付款申请：供应商付款、客户退款、借款、其他付款。
- 开票申请：来自 Altoc 合同或回款节点的开票申请。
- 审批编排：Finance 发起 Workflow 实例，Workflow 负责节点、待办和状态。
- 钉钉/企业微信对接：通过 Console/Integration 接入审批通知、待办提醒和审批结果同步。
- 审批结果落账：审批通过后生成支出、付款、报销或开票处理记录。

验收标准：

- 费用报销、项目支出、付款申请可发起审批。
- 审批状态在 Finance 中可追踪。
- 钉钉或企业微信可收到审批通知。
- 审批通过后自动进入对应财务台账。
- 外部平台凭证不写入 Finance 本地 env。

### 7.3 v0.3：项目核算与绩效金额财务口径

目标：基于 Altoc、Aims、People 和 Finance 数据，形成项目财务核算、人员成本分摊和绩效金额财务口径；个人绩效主流程由 People 负责。

范围：

- 项目收入：合同收款、非合同项目收入、收入归属。
- 项目支出：项目采购、外协、差旅、招待、销售费用、退款、手续费。
- 项目清单：以 Aims 项目事实为准，Finance 只合并财务摘要，不要求用户手工输入项目编码生成清单。
- 人力成本分摊：Finance 维护基本工资、福利费率、管理分摊系数和固定资源分摊等参数；People 维护员工职级和 M/P 职级工资、绩效工资范围；Finance 从 Aims 获取项目/员工月度工时，从 People 获取员工职级与职级设置，按标准成本实时计算项目人力成本。
- 项目毛利：收入、直接成本、人力成本、费用分摊、毛利和毛利率。
- 财务贡献归因：销售贡献、项目贡献、回款贡献、交付贡献等可量化财务输入。
- 绩效金额财务口径：提成、奖金、绩效金额或来自 People 的绩效评分输入所对应的财务快照。
- 管理看板：按项目、客户、部门、人员查看经营财务结果。

验收标准：

- 能按 Aims 项目输出收入、支出、成本状态和毛利；若 Aims 工时、People 员工职级/职级设置或 Finance 人力成本参数未就绪，必须标识为成本未就绪，不视为完整项目成本核算。
- 能按人员输出财务贡献、绩效金额快照和计算依据，供 People 绩效周期引用。
- People、Aims、Altoc 和 Finance 的引用关系清晰可追溯。
- 绩效金额财务口径可配置，并能保留计算快照；个人绩效确认和归档不在 Finance 内完成。

### 7.4 v0.4：应收应付、预算和现金流

目标：从台账和审批进一步升级为经营财务管理能力，让企业负责人、财务负责人、销售负责人和项目负责人能看清未来资金压力、预算占用和经营风险。

范围：

- 应收账款：按客户、合同、回款计划、发票和销售负责人统计应收、已收、未收、逾期和账龄。
- 应付账款：按供应商、项目、付款申请、采购/外协/费用类型统计应付、已付、未付和逾期。
- 资金计划：按 30/60/90 天滚动预测预计收款、预计付款、现金缺口和资金安全线。
- 预算管理：支持部门预算、项目预算、费用科目预算和预算周期。
- 预算控制：支持预算占用、释放、调整、超预算审批和预算执行分析。
- 经营看板：展示收入、回款、费用、毛利、应收、应付、现金流、项目利润排行和客户贡献排行。
- 经营预警：逾期应收、集中付款、现金流缺口、预算超支、项目毛利异常。

验收标准：

- 能按客户、合同和销售负责人输出应收账龄。
- 能按供应商、项目和费用类型输出应付计划。
- 能输出未来 30/60/90 天现金流预测。
- 部门、项目和费用科目预算可配置、占用、释放和分析。
- 企业负责人能在经营看板中看到资金风险和利润结果。

### 7.5 v0.5：财务自动化与准核算闭环

目标：降低财务人员重复录入和人工匹配成本，形成从流水、发票、单据、凭证草稿到月结快照的准核算闭环。

范围：

- 银行流水导入：支持 Excel/CSV/网银导出文件导入，识别交易时间、金额、对方户名、摘要和账户。
- 流水自动匹配：根据付款方、金额、合同、发票、回款计划和历史规则匹配客户回款或付款记录。
- 待处理流水池：无法自动匹配的流水进入待归类队列，支持人工确认和规则沉淀。
- 电子发票识别：识别发票号码、开票日期、购销方、金额、税额、价税合计和发票文件。
- 发票查重与归档：按发票号码、金额、购销方和文件指纹校验重复，关联报销、付款、项目成本或合同收入。
- 凭证草稿：根据开票、回款、报销、付款、费用归集生成凭证草稿和科目映射，不直接生成法定凭证。
- 专业财务软件导出：支持导出标准凭证草稿、发票、回款、费用和辅助核算维度，用于对接金蝶、用友、畅捷通等系统。
- 税务辅助：统计销项、进项、可抵扣、不可抵扣、税负和项目/合同税额拆分。
- 月结快照：按月锁定收入、费用、应收、应付、项目毛利和绩效金额财务口径快照，历史调整走调整记录。

验收标准：

- 银行流水导入成功率 >= 99%，重复导入可识别。
- 常见客户回款自动匹配率 >= 70%，未匹配流水可人工处理。
- 电子发票关键信息识别后可人工校正并归档。
- 凭证草稿可导出给外部专业财务软件。
- 月结快照可追溯，月结后核心历史数据不可直接改写。

## 9. 关键流程

### 9.1 合同开票与收款

1. Altoc 合同生效并生成经营回款计划。
2. 销售或财务从 Altoc 发起开票申请。
3. Finance 接收 `invoice_request`，发起审批。
4. 审批通过后，Finance 生成正式发票。
5. Finance 登记到账，核销到发票、合同或回款计划。
6. Finance 回传合同财务摘要给 Altoc。
7. Altoc 更新经营回款进度和应收风险。

### 9.2 费用报销

1. 员工提交费用报销单和明细。
2. Finance 根据费用类型、金额、部门、项目生成审批上下文。
3. Finance 调用 Workflow 发起审批。
4. Workflow 通过钉钉或企业微信通知审批人。
5. 审批通过后，Finance 生成支出记录。
6. 若归属项目，进入项目成本核算。

### 9.3 项目支出

1. 项目负责人或财务提交项目支出申请。
2. 选择项目、合同、供应商、支出类型和付款账户。
3. Workflow 完成审批。
4. Finance 生成项目支出台账。
5. v0.3 中进入项目毛利计算。

### 9.4 项目财务核算

1. Altoc 提供合同和客户。
2. Aims 提供项目、工时、成员和验收。
3. People 提供员工、岗位、职级和 M/P 职级设置；Aims 提供项目月度工时。
4. Finance 聚合收入、支出、人力标准成本和费用分摊。
5. Finance 生成项目毛利、财务贡献归因和绩效金额财务口径快照，供 People 个人绩效流程引用。

### 9.5 应收应付与现金流

1. Altoc 提供合同、回款计划和销售负责人。
2. Finance 聚合发票、回款、核销、付款申请和支出台账。
3. Finance 生成应收、应付和账龄快照。
4. Finance 按未来 30/60/90 天生成资金计划。
5. 经营看板展示现金流缺口、逾期应收、集中付款和项目利润风险。

### 9.6 预算控制

1. 财务负责人配置部门、项目或费用科目的预算周期和预算额度。
2. 报销、项目支出或付款申请提交时，Finance 生成预算占用。
3. 审批通过后预算占用转为实际支出。
4. 审批驳回或单据撤销后释放预算占用。
5. 超预算时触发 Workflow 的额外审批节点或风险提示。

### 9.7 银行流水与发票自动化

1. 财务上传银行流水或电子发票文件。
2. Finance 解析文件并生成导入批次。
3. Finance 根据金额、户名、合同、发票和历史规则自动匹配。
4. 未匹配记录进入待处理池，人工处理后沉淀匹配规则。
5. 开票、回款、报销和付款记录可生成凭证草稿。
6. 月末生成月结快照，锁定经营财务结果。

## 10. 外部集成

### 10.1 Workflow

Finance 不实现审批引擎，只定义财务单据和审批上下文。审批状态、节点、待办、回调由 Workflow 管理。

推荐动作：

- `finance.invoice.request`
- `finance.expense.claim`
- `finance.project_expense.request`
- `finance.payment.request`
- `finance.loan.request`

### 10.2 钉钉 / 企业微信

Finance 不直接保存钉钉或企业微信密钥。外部平台连接、凭证、机器人、OAuth、回调地址由 Console/Integration 统一管理。

Finance 需要的能力：

- 发起审批通知。
- 审批待办提醒。
- 审批结果回调。
- 单据状态变更通知。
- 支出/报销/付款提醒。

### 10.3 People

People 提供人和组织业务事实，Console Directory 提供登录身份和目录基础能力。Finance 只消费必要字段：

- 员工 ID、姓名、部门、岗位、职级。
- 入职、离职、在职状态。
- 成本中心。
- 员工职级、M/P 职级设置和项目/员工月度工时。

Finance 向 People 提供：

- 当前有效人力成本参数：基本工资、福利成本费率、管理分摊系数、固定资源分摊和币种。
- 绩效周期、贡献快照和必要的绩效评分输入仅用于个人绩效金额口径，不参与项目成本核算。

Finance 输出：

- 项目财务贡献归因。
- 绩效金额财务口径。
- 提成/奖金计算结果快照。
- 项目或部门成本归属。

约束：个人绩效周期创建、评分、确认、申诉和归档归 People；Finance 只提供可追溯的金额、成本和财务计算依据。

### 10.4 Altoc

Altoc 是合同和经营回款计划事实源。Finance 通过 API 获取或接收：

- `customerCode`
- `contractCode`
- `receivablePlanCode`
- 合同金额、付款条款、销售负责人。

Finance 回传：

- 开票金额。
- 到账金额。
- 核销金额。
- 未收金额。
- 最近到账日期。
- 应收风险状态。

### 10.5 Aims

Aims 是项目执行事实源。Finance 消费：

- 项目清单，作为项目核算页底表。
- 项目编码。
- 项目负责人。
- 项目成员。
- 工时。
- 里程碑和验收状态。

Finance 输出：

- 项目收入。
- 项目支出。
- 人力成本。
- 项目毛利。
- 成本异常提醒。

## 12. 数据迁移策略

### 12.1 从 wizbizdb 迁入 Finance

| 源表 | 目标 |
| --- | --- |
| `wb_invoice` | `finance_invoice` |
| `wb_project_income WHERE contract_id IS NOT NULL` | `finance_receipt` |
| `wb_project_income WHERE contract_id IS NULL` | `finance_unclassified_income` |
| `wb_project_payment` | `finance_expense` |
| `wb_bank_account` | `finance_bank_account` |
| `wb_account_balance` | `finance_account_balance_snapshot` |
| `wb_project_contribution` | v0.3 项目贡献导入源 |

### 12.2 与 Altoc 迁移的关系

- `wb_contract` 迁入 Altoc `contract`。
- Finance 通过 `contractCode` 或 `legacy_source + legacy_id` 关联 Altoc 合同。
- 发票、到账、支出长期不进入 Altoc。
- Altoc 的 `invoice/payment_record` 如已作为过渡表使用，需在 Finance 落地后清退或改为只读投影。

## 14. 风险与约束

1. 财务数据权限敏感，必须按角色和范围限制查看。
2. 外部审批平台状态可能和 Workflow 状态不一致，需要幂等回调和冲突处理。
3. People 数据如果不完整，会影响人力成本和绩效金额财务口径准确性。
4. `wizbizdb` 中历史财务字段存在口径差异，迁移时要保留源字段快照。
5. 不应过早实现完整总账，否则会显著扩大范围并拖慢经营财务主链路。
6. v0.5 的自动识别和自动匹配只能作为辅助能力，必须保留人工校正、审核和追溯。
7. 凭证草稿不是法定会计凭证，必须在产品文案和接口边界中明确。
